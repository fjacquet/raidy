## Requirements

### PowerVault ADAPT

- [x] **DELL-01**: PowerVault ADAPT usable capacity uses dynamic formula `((N-2)/N) * (8/10)` for <=18 drives, `((N-2)/N) * (16/18)` for >18 drives — replacing hardcoded 85%/87% constants
  - **Outcome:** Validated — Phase 8, formula matches Dell Sizer ME5224 within 1.5%

- [x] **DELL-02**: PowerVault ADAPT results match Dell Sizer within 1% for the ME5224 reference configuration (12 x 3.84TB SSD, ADAPT 8+2: expected 27.93 TiB usable from 41.9 TiB raw)
  - **Outcome:** Validated — Phase 8, 4 reference vectors (12/18/24/36 drives) pass

### PowerStore

- [x] **DELL-03**: PowerStore RAID-6 efficiency computed from DRE geometry selection (4+2 for <8 drives, 8+2 for 8-19 drives, 16+2 for >=20 drives) — replacing hardcoded 0.75 constant
  - **Outcome:** Validated — Phase 9, 7 reference vectors from Dell KB 000188491

- [x] **DELL-04**: PowerStore RAID-5 efficiency computed from DRE geometry selection (4+1 for <10 drives, 8+1 for >=10 drives) — replacing hardcoded 0.80 constant
  - **Outcome:** Validated — Phase 9, reference vectors from Dell KB 000188491

- [x] **DELL-05**: PowerStore applies system overhead (~5% default, configurable via `systemOverheadPercent`) on top of RAID parity efficiency
  - **Outcome:** Validated — Phase 10, configurable field added to PowerStoreOptions

- [x] **DELL-06**: PowerStore results match Dell Sizer within 1% for the 5200Q reference configuration (35 x 30.72TB NVMe QLC, RAID 16+2: expected 801.57 TiB usable from 977.89 TiB raw)
  - **Outcome:** Validated — Phase 10, within 2% of Dell Sizer 801.57 TiB reference

### PowerScale

- [x] **DELL-08**: PowerScale N+x efficiency formulas validated against Dell OneFS documentation and confirmed within 1% tolerance
