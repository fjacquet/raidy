# Milestone v1.2: Dell Calculation Accuracy

**Status:** SHIPPED 2026-03-25
**Phases:** 8-13
**Total Plans:** 7

## Overview

Fix Dell storage calculation formulas to match official Dell Sizer output within 1% tolerance across PowerVault ADAPT, PowerStore DRE, PowerScale OneFS, PowerFlex, and ObjectScale.

## Phases

### Phase 8: PowerVault ADAPT Formula Fix

**Goal**: Users calculating PowerVault ME5 capacity get accurate usable capacity based on their actual drive count
**Depends on**: Nothing (independent of other v1.2 phases)
**Plans**: 1 plan

Plans:

- [x] 08-01-PLAN.md — Fix ADAPT formula with Dell Sizer reference vectors and TDD test protocol

**Details:**
Dynamic ADAPT formula `((N-2)/N) * stripe_efficiency` replaces hardcoded 85%/87% constants, fixing a 28-percentage-point error for 12-drive configurations. Validated against Dell Sizer ME5224 (12x3.84TB SSD, ADAPT 8+2: 27.93 TiB usable from 41.9 TiB raw).

---

### Phase 9: PowerStore Data Fraction Fix

**Goal**: Users calculating PowerStore RAID-5 and RAID-6 capacity see drive-count-aware efficiency from the actual DRE geometry
**Depends on**: Nothing (independent; must precede Phase 10)
**Plans**: 1 plan

Plans:

- [x] 09-01-PLAN.md — Fix PowerStore DRE geometry with drive-count-aware RAID-5/RAID-6 efficiency and Dell KB reference vectors

**Details:**
PowerStore RAID-5/6 capacity calculations replaced hardcoded 0.75/0.80 constants with drive-count-aware DRE geometry from Dell KB 000188491. RAID-6: <8=4+2 (66.67%), 8-19=8+2 (80%), >=20=16+2 (88.89%). RAID-5: <10=4+1 (80%), >=10=8+1 (88.89%).

## Milestone Summary

**Key Decisions:**

- Dell Sizer as primary reference for all Dell platform calculations
- TDD protocol: skip wrong tests → add correct reference vectors (RED) → fix formula (GREEN) → delete skipped tests
- PowerVault ADAPT threshold at >18 drives (8+2 for <=18, 16+2 for >18) per Dell ME5 Admin Guide
- PowerStore DRE geometry from Dell KB 000188491 with 3 RAID-6 thresholds and 2 RAID-5 thresholds
- PowerStore system overhead 5% default, back-calculated from Dell Sizer 5200Q reference
- PowerScale OneFS protection is node-level: use serverCount not driveCount

**Issues Resolved:**

- PowerVault ADAPT: 28pp capacity error fixed (hardcoded 85% → dynamic 66.67% for 12 drives)
- PowerStore RAID-6: 5-22pp capacity error fixed (hardcoded 75% → drive-count-aware 66.67%-88.89%)
- PowerStore system overhead: missing deduction added (5% default matches Dell Sizer within 2%)
- PowerScale N+x: 19pp capacity error fixed (driveCount bug → serverCount)
- Test coverage: below 75% threshold → 84.13%

**Issues Deferred:**

- PowerStore system overhead per appliance model class (1000T vs 3200 vs 5200Q differ) — v2
- ObjectScale two-site geo-replication factor (2.67x overhead) — v2
- PowerScale small-file overhead modeling — v2
- PowerFlex FG metadata overhead per pool — v2
- Phases 5-6 (Performance & Fixes, Production Validation) from v1.0 — v1.3

**Technical Debt Incurred:**

- resilienceWorker.ts noNonNullAssertion warnings (pre-existing, not new)
- Some test files use `as any` for partial option objects in type assertions
