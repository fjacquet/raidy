    - RAID 6 with 5 drives, 1 hot spare (4 data drives = minimum)
    - Validate hot spares don't break capacity calculations

    Property-based tests with extreme values:
    - Use fast-check with fc.integer({ min: 100, max: 500 }) for drive counts
    - Use fc.integer({ min: 1e12, max: 20e12 }) for extreme capacities
    - Validate calculations always produce non-negative usable capacity
    - Validate efficiency is always between 0 and 1

    Test pattern:
    ```typescript
    it('should handle extreme RAID 6 with 200 drives correctly', () => {
      const input: VolumetryInput = {
        topology: { type: 'standard', level: 'RAID6' },
        driveCount: 200,
        drive: { ...testDrive, capacity_raw: 20e12 }, // 20TB
        serverCount: 1,

      const result = calculateVolumetry(input)

      expect(result.usableCapacity).toBeCloseTo(3960e12, -10) // Within 10 bytes tolerance
    ```
