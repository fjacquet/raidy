<task type="auto">
  <name>Update vitest.config.ts for React component testing</name>
  <files>vitest.config.ts</files>
  <action>
Update vitest.config.ts with the following changes based on RESEARCH.md recommendations:

**1. Change environment from 'node' to 'jsdom':**
```typescript
environment: 'jsdom',
```
This enables React component testing by providing browser globals (document, window, HTMLElement).

**2. Add setupFiles configuration:**
```typescript
setupFiles: ['./src/test/setup.ts'],
```
This will load jest-dom matchers and test cleanup (file created in next plan).

**3. Update coverage thresholds from 20% to 75%:**
```typescript
thresholds: {
  lines: 75,
  functions: 75,
  branches: 75,
  statements: 75,
},
```
Research recommends 75-80% for production-ready projects. Starting at 75% provides quality gate while remaining achievable.

**4. Keep existing configuration:**
- globals: true (allows describe/it/expect without imports)
- provider: 'v8' (fast coverage with AST remapping)
- include patterns (already correct for discovering test files)
- Path aliases (already configured, inherited from vite.config.ts)

**What NOT to change:**
- Do NOT remove existing include patterns
- Do NOT remove path alias configuration
- Do NOT change coverage.provider from v8
- Do NOT add coverage.all option (removed in Vitest 4.0)

The final test configuration should look like:

```typescript
test: {
  globals: true,
  environment: 'jsdom',  // CHANGED from 'node'
  setupFiles: ['./src/test/setup.ts'],  // ADDED
  include: ['src/**/*.{test,spec}.{ts,tsx}', 'tests/**/*.{test,spec}.{ts,tsx}'],
  coverage: {
    provider: 'v8',
    reporter: ['text', 'json', 'html'],
    include: ['src/engines/**/*.ts', 'src/workers/**/*.ts', 'src/utils/**/*.ts'],
    exclude: ['**/*.d.ts', '**/*.test.ts', '**/*.spec.ts'],
    thresholds: {
      lines: 75,      // CHANGED from 20
      functions: 75,  // CHANGED from 20
      branches: 75,   // CHANGED from 20
      statements: 75, // CHANGED from 20
    },
  },
},
```
  </action>
