Create src/test/ directory and setup.ts file with the following content:

```typescript
/**
 * Vitest Global Test Setup
 *
 * Provides:
 * - jest-dom custom matchers (toBeInTheDocument, toHaveClass, etc.)
 * - Automatic cleanup after each test
 */

import '@testing-library/jest-dom/vitest'
import { cleanup } from '@testing-library/react'
import { afterEach } from 'vitest'

// Automatic cleanup after each test to prevent memory leaks and state pollution
afterEach(() => {
  cleanup()
})
```

This setup file provides:
1. **jest-dom matchers globally:** Tests can use `expect(element).toBeInTheDocument()` without importing jest-dom
2. **Automatic cleanup:** React Testing Library cleanup runs after each test, preventing component state leakage between tests

Why this pattern (from RESEARCH.md):
- Centralizes test configuration instead of repeating imports in every test file
- Ensures cleanup happens even if tests fail or are skipped
- Provides semantic DOM matchers: toBeInTheDocument, toHaveClass, toBeDisabled, toHaveTextContent, etc.

Directory creation:
