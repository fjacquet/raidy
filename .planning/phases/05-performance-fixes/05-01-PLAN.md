<objective>
Split monolithic useCalculations hook into independent memoization hooks to prevent unnecessary recalculation.

Purpose: Currently, changing any configuration value (e.g., compressionRatio) triggers ALL three calculation engines (volumetry, performance, sustainability) to re-run because they share a single useMemo with 30+ dependencies. This wastes CPU cycles and degrades UI responsiveness. Splitting into independent hooks with focused dependency arrays ensures only affected calculations re-run.

Output: Three new hooks (useVolumetryCalc, usePerformanceCalc, useSustainabilityCalc) with focused dependencies, orchestrated by a lightweight useCalculations wrapper.
</objective>
