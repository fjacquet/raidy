<objective>
Fix PowerScale N+x capacity calculations to use serverCount (node count) instead of driveCount as the denominator in the OneFS protection formula.

Purpose: PowerScale protection overhead is node-level, not drive-level. The OneFS formula is `N/(N+M)` where N = data nodes and M = protection nodes. Currently the strategy receives driveCount (total drives in cluster), so a 10-node x 36-drive/node cluster computes (360-2)/360 = 99.4% efficiency instead of the correct (10-2)/10 = 80%. This is a 19-percentage-point error that causes massive over-provisioning estimates.

Output: Corrected PowerScale strategy using nodeCount from options, serverCount threading in calculationHelpers.ts, PowerScale reference vectors in dell-vectors.ts, passing test suite with multi-drive-per-node test cases.
</objective>
