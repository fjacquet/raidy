  <action>
Replace `any` type with `unknown` for options parameters in PerformanceStrategy interface.

**Current code (lines 34, 60):**

```typescript
getWritePenalty(level: string, options?: any): number
calculateIOPS(level: string, driveCount: number, driveIOPS: number, readPercent: number, options?: any): number
```

**Fix:**

```typescript
getWritePenalty(level: string, options?: unknown): number
calculateIOPS(level: string, driveCount: number, driveIOPS: number, readPercent: number, options?: unknown): number
```

**Why unknown instead of any:**

- `unknown` is type-safe: forces type checking before use
- `any` disables type checking entirely (unsafe)
- Biome noExplicitAny rule forbids `any` but allows `unknown`
- Options parameter is optional and topology-specific, so unknown is appropriate

**Alternative considered:**
Creating specific option types for each topology (ZfsOptions, CephOptions, etc.) would be more type-safe but requires significant interface changes. Using `unknown` is the minimal fix that satisfies the lint rule while maintaining backward compatibility.

After fix, run lint to verify:
