**File: src/hooks/useCalculations.ts**

Current structure: useCalculations calls volumetry, performance, sustainability engines. Phase 3 added validation gate that returns zero-state on blocking errors.

Add try/catch around each calculation engine call with context logging:

```typescript
// Around volumetry calculation
try {
  const volumetryResult = calculateVolumetry({
    drive,
    driveCount,
    hotSpares,
    topology,
    // ... other params
  })
  setVolumetry(volumetryResult)
} catch (error) {
  console.error('[Volumetry Engine Error]', {
    message: error instanceof Error ? error.message : 'Unknown error',
    context: {
      driveId: drive.id,
      driveCount,
      topology: topology.type,
      level: topology.level,
    },
    timestamp: new Date().toISOString(),
  })
  // Set safe fallback state
  setVolumetry({
    rawCapacity: 0,
    usableCapacity: 0,
    efficiency: 0,
    capacityBreakdown: [],
  })
}
```

Apply same pattern for:
- Performance engine calculation
- Sustainability engine calculation

**Why this approach:**
- Structured logging with context helps debug issues
- Includes configuration state that caused error (drive ID, topology, counts)
- Timestamp for log correlation
- Never exposes error.stack to users (security per Phase 3 decision)
- Sets safe fallback instead of crashing (graceful degradation)
