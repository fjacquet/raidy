Sanitize all user-controlled inputs in PDF export to prevent XSS vectors and ensure safe document generation.

Purpose: jsPDF processes user data for text rendering. While jsPDF 4.0.0 fixes CVE-2025-68428 path traversal, user-controlled text fields (projectName) could contain malicious HTML/scripts. DOMPurify sanitization ensures only plain text reaches PDF rendering.

Output: PDF export with sanitized inputs, passing SEC-07 (sanitization) and SEC-08 (XSS review) requirements.
</objective>

```typescript
/**
 * Sanitize user input for PDF rendering.
 * Strips all HTML tags, keeps text content only.
 */
function sanitizeForPdf(input: string): string {
  return DOMPurify.sanitize(input, {
    ALLOWED_TAGS: [],      // Strip all HTML
    KEEP_CONTENT: true     // Keep text content
  })
}
```

4. Apply sanitization before jsPDF rendering (line 73):
```typescript

doc.text(safeProjectName, pageWidth / 2, y, { align: 'center' })
```
