<objective>
Fix PowerStore RAID-5 and RAID-6 capacity calculations to use DRE drive-count-aware geometry instead of hardcoded constants.

Purpose: PowerStore currently returns 0.75 for all RAID-6 configs and 0.80 for all RAID-5 configs, regardless of drive count. The actual Dell DRE engine selects different stripe widths based on drive count (e.g., RAID-6 with 6 drives uses 4+2=66.7%, but with 20 drives uses 16+2=88.9%). This causes 5-22 percentage point errors in capacity calculations.

Output: Corrected dell.ts strategy with drive-count-aware DRE geometry, Dell KB reference test vectors in dell-vectors.ts, passing test suite.
</objective>
